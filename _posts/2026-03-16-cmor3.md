Bugfixes
* Improved error messages for time interval validation to provide clearer guidance to users. ([#915][pr-915]{:target="_blank"})
  * When a dataset has a defined `frequency` attribute, the error message will include both the frequency name and its corresponding approximate time interval.
* Added a clearer error message when Zstandard compression is requested but the ZSTD filter is unavailable. ([#925][pr-925]{:target="_blank"})
  * This error instructs users to install `hdf5plugin` and set the `HDF5_PLUGIN_PATH` environment variable.

Changes
* CMIP7 datasets: ignore the `branch_method` global attribute and do not add `table_id` to output files. ([#926][pr-926]{:target="_blank"})

CI / Tests
* CI build migrated from `conda-build` to `rattler-build`, improving build performance. ([#922][pr-922]{:target="_blank"})
* CI test environment now installs `hdf5plugin` and configures `HDF5_PLUGIN_PATH` for the NetCDF ZSTD filter. ([#922][pr-922]{:target="_blank"})
* Updated CMIP7 test file templates. ([#921][pr-921]{:target="_blank"})
* Updated CMIP7 dataset unit tests to match latest required/optional global attributes. ([#926][pr-926]{:target="_blank"})
  * Now requires `license_id`, adds `drs_specs`, no longer requires `experiment`, and removes `grid` and `source_type`.

* [Migrate CI build to conda-rattler (#919)][issue-919]{:target="_blank"}
* [Update error message for zstandard compression to warn users to set up HDF5_PLUGIN_PATH (#923)][issue-923]{:target="_blank"}
* [CMIP7: branch_method global attribute dropped for CMIP7 still required by CMOR (#924)][issue-924]{:target="_blank"}
